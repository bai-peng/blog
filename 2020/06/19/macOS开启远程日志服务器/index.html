macOS开启远程日志服务器

在macOS中开启日志服务器，允许来自远端的日志消息打印到系统控制台,可执行如下操作：

打开 UDP 日志接收端口：

查看 UDP 端口是否已经打开：

sudo lsof -i :514 -P

配置

编辑asl配置文件,推荐新建文件放到/etc/asl/ 目录下,

首先，需要准确地找到远程主机发送过来消息，消息写在Host和Facility字段中。为此，运行命令syslog -F raw，从命令输出中查看输出格式最后进行过滤。这里查看下面的示例输出;

vi /etc/asl/Mikrotik

添加如下内容；

? [= Host MikroTik] [= Facility local7] file /var/log/Mikrotik.log

需要说明的是,上面的[= Host MikroTik] [= Facility local7] 需要改成在syslog -F raw命令中输出的你的设备的Host和Facility,当然，也可以添加其他内容做为过滤。

重启服务使配置生效.

sudo launchctl unload /System/Library/LaunchDaemons/com.apple.syslogd.plist

sudo launchctl load /System/Library/LaunchDaemons/com.apple.syslogd.plist

以上测试在macOS 10.15.5 下测试通过。
